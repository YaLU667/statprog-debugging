At first, I was confused about the difference between Git and GitHub. I also found connecting my local repository to GitHub difficult because I did not fully understand what origin and branches meant.

The most useful thing I learned was the difference between git fetch and git pull. I learned that git fetch lets me inspect remote changes before merging them into my local repository, while git pull combines fetching and merging in one step.

I think I will use git fetch when working on shared projects where I want to review changes first before updating my local files.
